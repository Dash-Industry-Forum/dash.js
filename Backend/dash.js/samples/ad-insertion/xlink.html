XLink sample page

Working principle

Remote elements are elements that are not fully contained inside the manifest and can be resolved via XML Linking Language (XLink). Only a subset of the XLink speciﬁcation is needed for DASH. In order to use XLink, two attributes namely “xlink:href” and “xlink:actuate” have to be included inside an element. While the former is used to specify an URL to the remote content, the latter deﬁnes the resolution time. This page gives an overview of different XLink resolution procedures with the xlink:actuate attribute set to "onLoad". To start a demo please click on a testcase.
